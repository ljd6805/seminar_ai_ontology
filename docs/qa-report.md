# 75장 웹 교안 QA 보고서

- 대상: `slides/v2/`
- 기준일: 2026-07-22
- 기준 화면: 1366×768, 1600×900, 1920×1080
- 전체 학습 모드: 75장
- 발표 모드: 심화 15장을 제외한 60장

## 실제 브라우저 검수 결과

- 수정 전 공개본은 1366×768에서 27장, 1600×900에서 10장에 상·하단 잘림, 출처 겹침 또는 불필요한 스크롤이 있었다. 1920×1080에서는 문제가 발견되지 않았다.
- 수정 후 1366×768, 1600×900, 1920×1080에서 75장 전체의 요소 경계를 다시 검사했다. 세 해상도 모두 상·하단 잘림, 출처 겹침, 가로·세로 overflow가 0건이었다.
- 높이가 낮은 화면에서는 제목·간격·카드·도식의 크기를 함께 줄이는 압축 레이아웃이 적용되고, 콘텐츠가 더 길면 슬라이드 내부 세로 스크롤로 모든 내용을 열람할 수 있다.
- 본문과 제목에는 Pretendard Variable을, 코드·단축키·작은 레이블에는 JetBrains Mono를 적용했다. 두 글꼴을 불러오지 못할 때는 시스템 글꼴로 자연스럽게 대체된다.
- 내용이 밀집된 63번과 마무리 75번을 실제 화면으로 눈검사해 제목, 카드, 인용문, 출처가 안전 영역 안에 배치됨을 확인했다.
- 390×844 화면에서 가로 overflow가 없고, 긴 슬라이드는 세로 스크롤로 제목부터 출처까지 열람됨을 확인했다.
- 키보드 `←`/`→`, `PageUp`/`PageDown`, `Home`/`End`와 이전·다음 버튼이 정상 동작했다.
- 오버뷰 이동, 발표자 노트 열기/닫기, 애니메이션 다시 재생, 잘못된 hash 보정이 정상 동작했다.
- `?mode=presentation`은 심화 15장을 제외한 60장을, `?print=1`은 전체 75장을 펼쳐 표시했다.
- `prefers-reduced-motion` 환경에서는 단계 요소가 즉시 보이고 불필요한 전환이 억제됐다.

## 수동 인쇄 경로

PDF 산출물은 이번 작업에서 제외했다. 사용자는 필요할 때 `slides/?print=1`을 열어 브라우저의 **인쇄 → PDF로 저장** 기능을 사용할 수 있다. 인쇄용 CSS는 16:10 페이지와 전체 75장 펼침을 유지한다.

## 주장 경계

- 외부 이미지와 도표를 복제하지 않고, 출처에 근거한 설명용 도식을 새로 구성했다.
- 반도체 수치·이름·사건은 교육용 가상 데이터다.
- ontology가 RAG/Agent 정확도를 자동 향상시킨다는 인과 효과는 주장하지 않는다.
