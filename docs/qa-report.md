# 75장 웹 교안 QA 보고서

- 대상: `slides/v2/`
- 기준일: 2026-07-22
- 기준 화면: 1600×1000 (16:10)
- 전체 학습 모드: 75장
- 발표 모드: 심화 15장을 제외한 60장

## 검증 범위

| 항목 | 기준 | 결과 |
|---|---|---|
| 슬라이드 번호 | 1–75 연속, 중복·누락 없음 | 자동 테스트 통과 |
| 승인 제목 | `docs/seminar-page-list.md`와 일치 | 자동 테스트 통과 |
| 근거·노트 | 모든 화면에 source ID와 발표자 노트 존재 | 자동 테스트 통과 |
| 공개 진입점 | `slides/`가 `slides/v2/`로 query/hash를 보존해 이동 | 자동 테스트 통과 |
| 상호작용 | 이전/다음, 키보드, 오버뷰, 재생, 발표자 노트 | 실제 브라우저 검수 |
| 접근성 | skip link, live region, reduced motion, focus 가능한 제어 | 실제 브라우저 검수 |

| 반응형 | 데스크톱 16:10과 모바일 390×844 확인 | 실제 브라우저 검수 |
| 회귀 테스트 | `py -3 -m unittest discover -s tests -v` | 27개 테스트 통과 |

## 실제 브라우저 검수 결과

- 1600×1000 화면에서 75장을 모두 캡처하고 요소 경계를 검사했다. 잘림·겹침·뷰포트 이탈은 발견되지 않았다.
- 390×844 화면에서 가로 overflow가 없고, 긴 슬라이드는 세로 스크롤로 제목부터 출처까지 열람됨을 확인했다.
- 키보드 `←`/`→`, `PageUp`/`PageDown`, `Home`/`End`와 이전·다음 버튼이 정상 동작했다.
- 오버뷰 이동, 발표자 노트 열기/닫기, 애니메이션 다시 재생, 잘못된 hash 보정이 정상 동작했다.
- `?mode=presentation`은 심화 15장을 제외한 60장을, `?print=1`은 전체 75장을 펼쳐 표시했다.
- `prefers-reduced-motion` 환경에서는 단계 요소가 즉시 보이고 불필요한 전환이 억제됐다.

## 수동 인쇄 경로

PDF 산출물은 이번 작업에서 제외했다. 사용자는 필요할 때 `slides/?print=1`을 열어 브라우저의 **인쇄 → PDF로 저장** 기능을 사용할 수 있다. 인쇄용 CSS는 16:10 페이지와 전체 75장 펼침을 유지한다.

## 주장 경계

- 외부 이미지와 도표를 복제하지 않고, 출처에 근거한 설명용 도식을 새로 구성했다.
- 반도체 수치·이름·사건은 교육용 가상 데이터다.
- ontology가 RAG/Agent 정확도를 자동 향상시킨다는 인과 효과는 주장하지 않는다.
