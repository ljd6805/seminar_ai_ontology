# 교안 디자인 시스템 v3

## 디자인 방향

기준 저장소 [`seminar_ai_agent`](https://github.com/ljd6805/seminar_ai_agent)의 큰 결론형 제목, 섹션 오프너, 전체 오버뷰와 발표 모드를 계승한다. 새 덱은 이를 **엔지니어링 에디토리얼** 언어로 재구성한다. 16:9 안전 캔버스, 평면 레이어, 정밀한 연결선, 큰 타이포그래피를 중심으로 하며 유리 카드·과도한 라운드·장식용 네온은 사용하지 않는다.

웹 폰트가 차단되거나 오프라인이어도 시스템 폰트 대체 체계로 읽을 수 있어야 한다. 한글 UI와 본문은 `Pretendard Variable`, 코드·식별자·보조 레이블은 `JetBrains Mono`를 우선 사용한다.

## 의미 색 체계

- 개념과 연결: cyan `#51d5ee`
- 근거와 관측: teal `#45d4b4`
- 제약과 통과 조건: amber `#f1bd62`
- 위험과 실패: coral `#ff7f76`
- 거버넌스와 생명주기: violet `#aa8cff`

한 슬라이드에는 주 강조색 하나만 사용한다. 색만으로 상태나 관계를 구분하지 않고 선 종류, 관계 레이블, 상태 배지를 함께 제공한다. 작은 텍스트는 4.5:1, 큰 텍스트와 핵심 도형은 3:1 이상의 대비를 목표로 한다.

## 아홉 가지 마스터 레이아웃

1. 대형 제목과 핵심 그래프를 결합한 표지·섹션 오프너
2. 한 문장 주장과 하나의 초점 도식
3. 전·후 또는 두 개념을 대비하는 분할 화면
4. 레이어와 시스템 경계를 보여 주는 아키텍처 지도
5. 원인·절차·생명주기를 설명하는 흐름·타임라인
6. 방법론·요구사항을 비교하는 매트릭스·결정 지도
7. 코드와 그래프 요소를 동기화한 코드 워크스루
8. 실제 사례의 증거와 운영 구조를 보여 주는 사례 화면
9. 학습자가 직접 작성하는 워크시트·행동 화면

카드 그리드를 기본값으로 사용하지 않는다. 카드형 화면은 전체 10장 이하, 연속 2장 이하로 제한하고 `docs/seminar-page-list.md`의 “시각·애니메이션” 열을 슬라이드별 구현 정본으로 삼는다.

## 하이브리드 단계 공개

- 슬라이드 진입은 180–320ms의 짧은 페이드와 미세 이동만 자동 재생한다.
- `Space`·`→`는 현재 의미 단계를 하나씩 공개하고 마지막 단계 뒤 다음 슬라이드로 이동한다.
- `←`는 현재 의미 단계를 되돌리고 첫 단계 앞에서 이전 슬라이드로 이동한다.
- `R`은 현재 슬라이드의 의미 단계를 처음으로 되돌린다.
- `O` 또는 `Esc`는 오버뷰, `N`은 발표자 노트, `Home`·`End`는 처음·끝 이동에 사용한다.
- `?mode=presentation`은 부록을 제외하고 `?motion=off`는 이동 애니메이션 없이 상태를 즉시 바꾼다.
- `prefers-reduced-motion`, 인쇄, 오버뷰에서는 의미 요소의 최종 상태를 표시한다.

모션은 `trace`, `focus`, `transform`, `validate`, `result`만 사용한다. 다음 입력이 진행 중 효과를 즉시 완료하므로 발표자가 애니메이션을 기다릴 필요가 없다. 정지 상태에서도 같은 관계와 결과를 읽을 수 있어야 한다.

## 화면 안전 규칙

- 데스크톱은 16:9 캔버스를 중앙에 고정하고 남는 영역은 레터박스로 처리한다.
- 1366×768, 1600×900, 1920×1080에서 제목·도식·출처가 캔버스 경계를 넘지 않아야 한다.
- 모바일에서는 단일 열로 재배치하고 슬라이드 내부를 세로 탐색할 수 있게 한다. 가로 overflow는 허용하지 않는다.
- 리드 문장은 최대 두 줄, 본문은 최대 세 정보 묶음을 원칙으로 한다. 세부 설명은 발표자 노트로 이동한다.
- 의미형 SVG는 `role="img"`, 한국어 요약, 관계 방향, 관계 이름, 범례를 제공한다.

## 배포·호환성

기존 `window.OntologyDeck.register()`와 `#/번호` URL을 유지한다. 슬라이드 데이터는 `visualType`, `builds`, `motion`, `ariaSummary`를 선택적으로 선언하며, 기존 콘텐츠는 전환 기간 동안 평면형 호환 스타일로 렌더링한다. PDF 제작은 이 개편 범위에서 제외한다.
