# 교안 디자인 시스템

## 차용한 형식

기준 저장소 [`seminar_ai_agent`](https://github.com/ljd6805/seminar_ai_agent)의 학습 경험을 차용한다. 핵심은 16:10 화면, 진입 시 자동 스태거, 전체 오버뷰, 발표 모드, 키보드 이동, 진한 우주색 배경과 보라·민트 강조색이다. 주제와 본문 구조는 온톨로지 지식관리 교안에 맞게 새로 작성한다. 웹 폰트가 차단되거나 오프라인이어도 시스템 폰트 대체 체계로 콘텐츠를 읽을 수 있어야 한다.

## 시각 언어

- 배경: `#070917`을 중심으로 한 어두운 네이비와 낮은 대비의 별 격자
- 강조: 의미·검증은 민트, 구조·추론은 보라, 주의는 앰버와 코럴
- 타이포그래피: 한글 본문과 제목은 `Pretendard Variable`, 코드·식별자·보조 라벨은 `JetBrains Mono`를 우선 사용한다. 큰 제목, 한 화면 한 문장, 충분한 행간과 여백을 유지한다.
- 도식: 관계 그래프, 단계 흐름, 비교, 타임라인을 학습 목표가 있을 때만 사용
- 출처: 모든 본문 화면 하단에 근거 ID 또는 프로젝트 결정 ID 표시

## 상호작용

- 처음 보는 화면의 핵심 요소는 학습 순서대로 한 번 자동 재생한다.
- `재생` 버튼 또는 `R` 키로 현재 화면을 다시 재생한다.
- `←/→`, `PageUp/PageDown`, `Home/End` 키를 지원한다.
- `O` 또는 `Esc`로 오버뷰를 열고 닫는다.
- `N`으로 발표자 노트를 토글한다.
- `?mode=presentation`은 부록을 제외하고, `?print=1`은 모든 화면을 인쇄 배치로 전환한다.
- `prefers-reduced-motion` 사용자는 애니메이션 없이 모든 정보를 즉시 본다.

## 템플릿 규칙

각 화면은 학습 목표 하나만 다룬다. 제목은 결론형 문장으로 쓰고, 본문은 최대 두 개의 시각 그룹으로 제한한다. 작은 카드의 반복보다 큰 도식 하나, 비교 두 칸, 단계 흐름 중 가장 적합한 구성을 선택한다.

## 화면 높이 안전 규칙

- 16:10뿐 아니라 16:9 노트북 화면을 기준 화면으로 함께 검수한다.
- 제목·본문·도식의 크기는 화면 너비와 높이 중 더 엄격한 조건에 맞춰 줄어든다.
- 활성 슬라이드는 안전 중앙 정렬을 사용하고, 내용이 길면 슬라이드 내부에서만 세로 스크롤한다.
- 하단 출처 줄과 탐색 버튼이 본문을 가리지 않도록 화면 높이 900px 이하에서 간격과 도식 높이를 별도로 압축한다.
